## 接口说明
http/json 数据格式，其中回应的协议如下格式:
```json
{
    err_code : 0 | -1 | -2,
    err_desc : "错误描述",
    data : [] | {} //可选的data域，可以为数组或对象
}

```
> 业务逻辑无异常，err_code 为 0,有异常，填充非零值

> data域根据业务逻辑自行填充

> http Statuscode 总是 200

### 外部接口
#### 获取预约录制列表
请求URL:

GET http://ip:port/api/file

响应:
```json
{
  err_code : 0,
  data : [
    {
      _id : 数据唯一,
      name : "录制的文件名"，
      length : 13131, //文件长度
      start : "2016-05-01 12:13:14",
      end   : "2016-05-01 12:13:14"
    }
  ]
}
```
用户可根据文件名 拼接 服务器前缀即可正常访问
